Image Crawler Fork-Join Pool App (Assignment 2 Final Submission)

The purpose of this assignment is to deepen your experience using Java 8 functional programming features together with Java 8 sequential streams and their integration with the Java fork-join pool framework. In particular, in this assignment you'll improve the implementation of the app from Assignments 2a and 2b that uses the Java fork-join pool framework to obtain, store, and transform images.

Like Assignments 2a and 2b, the code you'll write here will have GUI and will run as an app. Please use this GUI along with the JUnit tests discussed below to help test and debug your solution.

Overview of the Assignment 2b

The bulk of the code for the app is provided in the skeletons. Your efforts will focus on the following tasks:

  • Implement the methods in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/UnsynchronizedArray.java file. The bulk of the methods in this class can be copied from your Array.java implementation from previous assignments. However, there are a few changes. For example, the latest Array.java file has been converted from a class into an interface to enhance flexibility. You'll therefore find it necessary to implement and use several new accessor methods for various fields previously defined in the Array class.

  • Complete the collector implementation in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/ArrayCollector.java file. Graduate students must implement the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/SynchronizedArray class implementation and use it in their ArrayCollector.java implementation. Undergraduate students can ignore the SynchronizedArray class and use the UnsynchronizedArray class.

  • Complete the image crawler class implementation in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/ForkJoinCrawler1.java and image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/ForkJoinCrawler2.java files, which uses the Java fork-join pool framework to perform an "image crawl" starting from a root Uri. All students must use Java 8 lambda expressions, method references, and sequential streams for their ForkJoinCrawler2.java implementation.

  • Undergraduates will need to implement the ArraySpliterator in UnsynchronizedArray (graduate students did this as part of a previous assignment).

JUnit Tests

Extensive JUnit tests in the image-crawler/src/test/java/edu/vanderbilt/imagecrawler folder are provided to increase our confidence that your implementation is working as expected. You can run all the unit tests by right-clicking on crawlers and utils in this folder and running them.

If you need help learning how to use the Android Studio GUI to run the JUnit tests locally on your computer please see here and/or ask questions. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the JUnit tests to detect problems in your code.
