Image Crawler Fork-Join Pool App (Assignment 2b)

The purpose of this assignment is to deepen your experience using Java 8 functional programming features together with Java 8 sequential streams and their integration with the Java fork-join pool. In particular, in this assignment you'll improve the implementation of the app from Assignment 2a that uses the Java fork-join pool framework to obtain, store, and transform images.

Like assignment 1, the code you'll write for this assignment will have GUI and will run as an app. Please use this GUI along with the unit tests to help test and debug your solution.

Overview of the Assignment 2b

The bulk of the code for the app is provided in the skeletons. Your efforts will focus on the following tasks:

  • Implement the methods in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/UnsynchronizedArray.java file. The bulk of the methods in this class can be copied from your Array.java implementation from previous assignments. Note that the latest Array.java file has been converted from a class into an interface to enhance flexibility.

  • Complete the collector implementation in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/ArrayCollector.java file.

  • Graduate students must implement the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/SynchronizedArray class implementation and use it in their ArrayCollector.java implementation. Undergraduate students can ignore the SynchronizedArray class.

  • Complete the image crawler class implementation in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/ForkJoinCrawler2.java file to obtain, store, and transform images. This class uses Java fork-join pool framework to perform an "image crawl" starting from a root Uri. All students must use Java 8 lambda expressions, method references, and sequential streams for their ForkJoinCrawler2.java implementation.

  • Refactor certain portions of your image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/ForkJoinCrawler1.java code from assignment 2a so we can auto-grade it accurately. Please read the comments carefully and make sure to put your code in the appropriate "graduate" vs. "undergraduate" branches.

Unit Tests

JUnit unit tests in the image-crawler/src/test/java/edu/vanderbilt/imagecrawler folder are provided to increase our confidence that your implementation is working as expected. You can run all the unit tests by right-clicking on 'assignmentTests' in this folder and running it.

If you need help learning how to use the Android Studio GUI to run the JUnit test locally on your computer please see here and/or ask questions. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the JUnit tests to detect problems in your code.

Concluding Remarks

The fork-join pool framework in Assignment 2b is designed to give you experience programming directly to the Java fork-join pool framework APIs using Java 8 sequential streams. There are more "moving parts" in this assignment compared with previous assignments. Therefore, please start the assignment soon and ask questions in class, office hours, and on the discussion forums. The app, image crawler skeletons, and JUnit tests are extensive, though you don't need to understand them all to complete your solution.
