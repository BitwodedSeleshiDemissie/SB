Image Crawler Parallel Stream App (Assignment 3)

The purpose of this assignment is to deepen your experience using Java 8 functional programming features and their integration with Java 8 parallel streams. In particular, in this assignment you'll develop an app that uses the Java 8 streams framework to obtain, transform, and store images.

Overview of the Assignment 3

In this part of the assignment you'll complete the implementation of an image crawler class in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/ParallelStreamsCrawler.java file that obtains, processes, and stores images. This class uses Java 8 parallel streams to perform an "image crawl" starting from a root Uri. Depending on the parameters used to run the tests, images from HTML page reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in files.

The bulk of the code for the app is provided in the skeletons. In fact, a complete solution using Java 7 features provided in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/SequentialLoopsCrawler.java file. Your will therefore focus on converting this Java 7-based class to the ParallelStreamsCrawler class that uses Java 8 parallel streams.

Everyone taking the course (i.e., both ugrads and grads) must also complete implementations of all the methods in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/ArrayCollector.java, image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/UnsynchronizedArray.java, and image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/SynchronizedArray.java files.

Your solution should use no loops in this assignment and instead use Java 8 parallel stream aggregate operations.

Skeleton Code

Skeleton code for this assignment is available here in my GitHub account. Now that you've setup your GitLab account you can pull this skeleton code in your repository, read it carefully, and complete the "TODO" markers.

Unit Tests

The JUnit unit tests in the image-crawler/src/test/java/edu/vanderbilt/imagecrawler folder are provided to increase our confidence that your implementation is working as expected. You can run all the unit tests by right-clicking on crawlers and utils in this folder and running them.

If you need help learning how to use the Android Studio GUI to run the JUnit test locally on your computer please see here and/or ask questions. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the JUnit tests to detect problems in your code.

Concluding Remarks

Once you've got Assignment 2 working properly (and address all my reviewer comments) it should be relatively straightforward to implement Assignment 3. This assignment is designed to deepen your experience developing apps using Java 8 functional programming features and Java 8 parallel streams without the need to program the fork-join thread pool directly. The skeletons and JUnit tests are extensive, though you don't need to understand them all to complete your solution. Please ask questions in class, office hours, and on the discussion forums.
