Image Crawler Fork-Join Pool App (Assignment 2a)

The purpose of this assignment is to deepen your experience using Java 8 functional programming features and their integration with the Java fork-join pool. In particular, in this assignment you'll develop an app that uses the Java fork-join pool framework to obtain, store, and transform images.

Unlike assignment 1, the code you'll write for this assignment will have GUI and will run as an app. Please use this GUI to help debug your solution.

Getting Started

Overview of the Assignment 2a

In this part of the assignment you'll complete the implementation of an image crawler class in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/ForkJoinCrawler.java file that obtains, stores, and transforms images. This class uses Java fork-join pool framework to perform an "image crawl" starting from a root Uri. Depending on the parameters used to run the tests, images from HTML page reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in files.

The bulk of the code for the app is provided in the skeletons. In fact, a complete solution using Java 7 features provided in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/SequentialLoopsCrawler.java file. You will therefore focus on converting this Java 7-based class to the ForkJoinCrawler class that uses the Java fork-join pool framework.

Everyone taking the course (i.e., both ugrads and grads) must also include your completed implementations of all the methods in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/Array.java file. Make sure that this class is implemented properly or your solution will not work and you'll lose points.

Undergraduates can either use conventional for Java 7 for-loop loops or Java 8 lambda expressions and method references, but graduate students must use Java 8 lambda expressions and method references whereever it makes sense.

Skeleton Code

Unit Tests

JUnit unit tests in the image-crawler/src/test/java/edu/vanderbilt/imagecrawler folder are provided to increase our confidence that your implementation is working as expected. You can run all the unit tests by right-clicking on 'AllTests' in this folder.

If you need help learning how to use the Android Studio GUI to run the JUnit test locally on your computer please see here and/or ask questions. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the JUnit tests to detect problems in your code.

Concluding Remarks

The fork-join pool framework in Assignment 2a is designed to give you experience programming directly to the Java fork-join pool framework APIs. Although this framework is powerful, you'll have to keep track of various details manually. Therefore, please start the assignment soon and ask questions in class, office hours, and on the discussion forums. The app, image crawler skeletons, and JUnit tests are extensive, though you don't need to understand them all to complete your solution.
