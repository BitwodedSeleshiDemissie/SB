Array (Assignment 1b)

The purpose of this assignment is to deeper your experience using Android Studio to develop and test a Java Array class written using Java 8 functional programming features, such as functional interfaces. Although we'll be using Android Studio, the code you'll write for this assignment will not have GUI and will not run as an app. It's important that your Array implementation work properly since it's required by all subsequent assignments in this course, including the ones that are Android apps.

Getting Started

Before you start with Assignment 1b you'll need to do the following things:

  • Make sure that you've installed Java 8. Instructions for doing this are available here.
  • Complete your implementation of assignment 1a.

Once you get my comments back your assignment 1a submission make sure to incorporate any changes into your assignment 1b solution.

Overview of the Array Assignment

An Array is an abstract data type (ADT) implemented via a single contiguous buffer that provides operations like get(), set(), add(), remove(), and size(). This class differs from Java built-in arrays in various ways, e.g., it can be resized and used in conjunction with other Java Collection Framework classes. Your task in Assignment 1b is to implement a generic Array class in Java that includes methods (such as replaceAll() and forEach()) using Java 8 features, such as functional interfaces.

Graduate students (or students who are taking this course for graduate credit) must do the following additional work:

  • Implement the requirements of the Spliterator interface. Please see the videos here and here for more information on Java Spliterators.

As much as possible, you should avoid using loops in this assignment and instead try to use Arrays.copyOf() and/or System.arraycopy() (this isn't possible for all parts of the code, of course).

The following resources may be helpful in completing this assignment:

  • java.utils.Arrays documentation.
  • The Oracle Java Tutorial

Unit Tests

JUnit unit tests in the app/src/test/java/edu/vandy/imagecrawler folder are provided to increase our confidence that your implementation is working as expected. You'll need to learn how to use the Android Studio GUI to run the JUnit test locally on your computer, as described here. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the JUnit tests to detect problems in your code.
